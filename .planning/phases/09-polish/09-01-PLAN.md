## Context

### Current ThinkingBubble State (Phase 8)

`ThinkingBubble` composable at line 1546 renders when `!isUser && thinkingText.isNotEmpty()`. The thinking text streams in real-time via `onThinking` callback. However:

1. **No fade-out animation** — when `onDone` fires, the thinking bubble disappears instantly (no `AnimatedVisibility` wrapping the conditional render)
2. **No state cleanup** — `thinkingText` field is never cleared after `onDone` or `onError`, leaving stale state if message is edited later
3. **Scroll anchoring unclear** — LazyColumn may jump when thinking bubble appears/disappears during streaming

### THINK-10: mainHandler.post{}

All 6 `onThinking` callbacks already use `mainHandler.post {}` for UI thread safety. This is satisfied.

### THINK-11: Scroll Anchoring

The thinking bubble grows in height as streaming text accumulates (up to 4 lines clamped). This can cause the response text below to shift, creating a "jumping" effect. Need to reserve space or use `animateContentSize` to smooth the transition.

### THINK-12: State Cleanup

`onDone` callback receives no thinking text parameter — it only marks completion. When thinking ends:
- The thinking bubble should fade out (`AnimatedVisibility`)
- The `thinkingText` should be cleared from the message to prevent stale state

## Tasks

### Task 1: Add AnimatedVisibility fade-out to thinking bubble

**File:** `app/src/main/java/com/gemma/gpuchat/MainActivity.kt`

**Action:**
Wrap the `ThinkingBubble` render condition in `AnimatedVisibility` with `FadeOut` transition (300ms). The thinking bubble should fade out when the thinking state ends.

**Current (line ~974-981):**
```kotlin
items(messages) { message ->
    Column {
        if (!message.isUser && message.thinkingText.isNotEmpty()) {
            ThinkingBubble(thinkingText = message.thinkingText)
        }
        ChatBubble(...)
    }
}
```

**After:**
```kotlin
items(messages) { message ->
    Column(modifier = Modifier.animateContentSize()) {
        AnimatedVisibility(
            visible = !message.isUser && message.thinkingText.isNotEmpty(),
            enter = fadeIn(animationSpec = tween(200)),
            exit = fadeOut(animationSpec = tween(300))
        ) {
            ThinkingBubble(thinkingText = message.thinkingText)
        }
        ChatBubble(...)
    }
}
```

**Verify:** `grep -c "AnimatedVisibility" app/src/main/java/com/gemma/gpuchat/MainActivity.kt` returns `1+`

**Done:** Thinking bubble fades out smoothly when response completes (visible=false condition met).

### Task 2: Clear thinkingText on onDone

**File:** `app/src/main/java/com/gemma/gpuchat/MainActivity.kt`

**Action:**
In each `onDone` callback across all 6 call sites, add logic to clear `thinkingText` from the last bot message after the done processing completes. This ensures the thinking bubble disappears after the fade-out.

**Pattern — add at end of each onDone callback:**
```kotlin
onDone = {
    // ... existing done logic ...
    mainHandler.post {
        // Clear thinkingText to trigger fade-out
        val lastBot = messages.indexOfLast { !it.isUser }
        if (lastBot >= 0) {
            messages = messages.mapIndexed { idx, msg ->
                if (idx == lastBot) msg.copy(thinkingText = "") else msg
            }
        }
    }
}
```

Apply to all 6 `sendMessage()` call sites (lines 403, 632, 679, 1075, 1246, 1370).

**Verify:** `grep -c "thinkingText = \\\"\\\"" app/src/main/java/com/gemma/gpuchat/MainActivity.kt` returns `6+`

**Done:** All 6 `onDone` callbacks clear `thinkingText` so the thinking bubble disappears after fade-out.

### Task 3: Clear thinkingText on onError

**File:** `app/src/main/java/com/gemma/gpuchat/MainActivity.kt`

**Action:**
In each `onError` callback across all 6 call sites, ensure `thinkingText` is cleared. This prevents orphaned thinking bubbles when errors occur.

**Pattern — add in error handler mainHandler.post block:**
```kotlin
onError = { error ->
    AppLogger.e(TAG, "Error: ${error.message}", error)
    mainHandler.post {
        isResponding = false
        // ... existing error handling ...
        // Clear thinkingText on error
        val lastBot = messages.indexOfLast { !it.isUser }
        if (lastBot >= 0) {
            messages = messages.mapIndexed { idx, msg ->
                if (idx == lastBot) msg.copy(thinkingText = "") else msg
            }
        }
    }
}
```

Apply to all 6 `sendMessage()` call sites. Note: some call sites may already handle error state fully — only add if `thinkingText` clearing is missing.

**Verify:** `grep -c "thinkingText = \\\"\\\"" app/src/main/java/com/gemma/gpuchat/MainActivity.kt` returns `6+` (same check as Task 2, covers both done and error)

**Done:** Error paths clear `thinkingText` so no orphaned bubble appears.
