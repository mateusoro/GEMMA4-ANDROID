### Task 2: Add onThinking parameter to sendMessage()

**File:** `app/src/main/java/com/gemma/gpuchat/LlmChatModelHelper.kt`

**Action:**
Update `sendMessage()` signature to accept `onThinking: ((String) -> Unit)? = null` as the 6th parameter (after `extraContext`). Then wire the callback at the thinking channel extraction point (lines 328-331).

**Signature change at line 264:**

```kotlin
fun sendMessage(
    message: String,
    onToken: (String) -> Unit,
    onDone: () -> Unit,
    onError: (Throwable) -> Unit,
    extraContext: Map<String, String> = emptyMap(),
    onThinking: ((String) -> Unit)? = null  // THINK-06: nullable callback for thought channel tokens
)
```

**Callback wiring at lines 328-331 (existing code):**

Replace:
```kotlin
val thinkingContent = message.channels["thought"]
if (!thinkingContent.isNullOrEmpty()) {
    AppLogger.d(TAG, "[THOUGHT-CHANNEL] $thinkingContent")
}
```

With:
```kotlin
val thinkingContent = message.channels["thought"]
if (!thinkingContent.isNullOrEmpty()) {
    AppLogger.d(TAG, "[THOUGHT-CHANNEL] $thinkingContent")
    onThinking?.invoke(thinkingContent)
}
```

**Verify:** `grep -n "onThinking" app/src/main/java/com/gemma/gpuchat/LlmChatModelHelper.kt | head -10` — parameter in signature and invoke call present

**Done:** `sendMessage()` accepts `onThinking: ((String) -> Unit)? = null`, and thinking tokens are delivered via `onThinking?.invoke()` without blocking UI thread.
